以下程式碼，會在 ESP32 板子上的 LED **亮 3 秒、滅 1 秒**，不斷重複。

```cpp
#include <Arduino.h>

#define LED_PIN 2

void setup() {
  pinMode(LED_PIN， OUTPUT);
}

void loop() {
  digitalWrite(LED_PIN， HIGH);
  delay(3000);
  digitalWrite(LED_PIN， LOW);
  delay(1000);
}
```

---

## 一、逐行解釋程式碼

### `#include <Arduino.h>`

把 Arduino 的工具箱引入程式。`pinMode`、`digitalWrite`、`delay` 這些函式都定義在裡面，少了這行它們全部無法使用。

- 在 **Arduino IDE** 開發:IDE 會在編譯前自動幫你加，所以看不到這行。
- 在 **Cursor + PlatformIO** 開發:走標準 C++ 編譯流程，沒有這個魔法，**必須自己手寫在第一行**，否則會出現 `'pinMode' was not declared` 之類的錯誤。

### `#define LED_PIN 2`

**巨集定義**:告訴編譯器「編譯前，把程式裡所有的 `LED_PIN` 文字替換成 `2`」。

- 它是純文字替換，不是變數、不佔記憶體，結尾不用分號。
- `2` 指的是 **GPIO 2** 這支腳位，大多數 ESP32 開發板的內建藍色 LED 焊在這支腳上。
- 不用 `#define` 也可以，直接寫 `pinMode(2， OUTPUT)` 一樣能跑。但直接寫死的數字叫「魔術數字」(magic number)，別人看不懂 2 是什麼意思。

> 補充:現代 C++ 更推薦寫 `const int LED_PIN = 2;`，有型別檢查更安全，效果相同。

### `void setup()`

**開機時只執行一次**的函式，負責初始化設定。例如設定腳位模式、開啟序列埠。

### `void loop()`

setup 跑完後，**無限重複執行**的函式，主要邏輯都寫在這裡。ESP32 程式必定有 setup 和 loop 這兩個函式。

### `pinMode(LED_PIN， OUTPUT)`

**設定腳位的工作方向**。每支 GPIO 使用前都要先宣告它是要輸出還是輸入。

| 模式     | 意思 | 用途                                             | 比喻             |
| -------- | ---- | ------------------------------------------------ | ---------------- |
| `OUTPUT` | 輸出 | 由 ESP32 主動送電出去:點 LED、驅動蜂鳴器、繼電器 | 把門設成「出口」 |
| `INPUT`  | 輸入 | 讀取外部送進來的訊號:按鈕、感測器                | 把門設成「入口」 |

門要先決定方向才能用，所以 `pinMode` 寫在 `setup()` 裡，開機設定一次即可。

⚠️ 忘記 pinMode 就直接 digitalWrite，程式不會報錯，但 LED 可能不亮或很微弱 — 初學者常見的「燈為什麼不亮」原因。

### `digitalWrite(LED_PIN， HIGH / LOW)`

控制腳位輸出的**電壓**:

- `HIGH` = 輸出 **3.3V**(高電位)→ 燈亮
- `LOW` = 輸出 **0V**(低電位)→ 燈滅

簡記:**HIGH = 開，LOW = 關**。(背後原理見下方「電學觀念」)

### `delay(毫秒)`

讓程式**暫停指定的毫秒數**，期間維持目前狀態。

- `delay(3000)` = 停 3 秒 → 所以燈亮 3 秒
- `delay(1000)` = 停 1 秒 → 所以燈滅 1 秒

改這兩個數字就能改變閃爍節奏。

### 三兄弟函式總整理

| 函式                           | 作用          | 前提           |
| ------------------------------ | ------------- | -------------- |
| `pinMode(pin， 模式)`          | 設定腳位方向  | 使用腳位前必做 |
| `digitalWrite(pin， HIGH/LOW)` | 輸出高/低電位 | 先設 OUTPUT    |
| `digitalRead(pin)`             | 讀取高/低電位 | 先設 INPUT     |

理解完後，回頭看補上註解的程式碼：

```cpp
#include <Arduino.h>   // 引入 Arduino 工具箱(pinMode、digitalWrite、delay 都在裡面)
                       // Arduino IDE 會自動加,PlatformIO 必須手寫

#define LED_PIN 2      // 給 GPIO 2 取名叫 LED_PIN(板載藍色 LED 接在這支腳位)
                       // 編譯前會把程式裡所有 LED_PIN 替換成 2

void setup() {                    // 開機時只執行一次:做初始化
  pinMode(LED_PIN, OUTPUT);       // 把 GPIO 2 設成「輸出」模式(打算送電出去)
}

void loop() {                     // setup 跑完後,無限重複執行
  digitalWrite(LED_PIN, HIGH);    // 輸出高電位 3.3V → LED 亮
  delay(3000);                    // 暫停 3000 毫秒(3 秒)→ 亮著維持 3 秒
  digitalWrite(LED_PIN, LOW);     // 輸出低電位 0V → LED 滅
  delay(1000);                    // 暫停 1000 毫秒(1 秒)→ 滅著維持 1 秒
}                                 // 回到 loop 開頭,繼續重複 → 燈就不斷閃爍
```

## 二、電學觀念(用水管比喻)

把電想成水，一切都好懂:

| 名詞                    | 水的比喻           | 說明                                                           |
| ----------------------- | ------------------ | -------------------------------------------------------------- |
| **電壓** (Voltage， V)  | 水壓               | 壓力越高，推力越強。ESP32 全身用 3.3V 工作                     |
| **電流** (Current)      | 水流               | 真的有電在流動才叫電流。**LED 是靠電流流過才亮**               |
| **高電位 / 低電位**     | 水壓高 / 低        | 就是 HIGH(3.3V)和 LOW(0V)的中文說法。「拉高 / 拉低」也是指這個 |
| **GND** (Ground， 接地) | 排水孔             | 永遠 0V。所有電流最後都要流回 GND 繞成一圈 — 所以叫「電路」    |
| **電壓差** (電位差)     | 兩點的水壓落差     | **有落差，水才會流。這是整個電路的核心**                       |
| **電阻** (Resistor)     | 捏細的水管         | 讓水流變小，保護脆弱的零件                                     |
| **GPIO**                | 可程式控制的水龍頭 | 板上那排有編號的腳位，每支都能當輸出或輸入用                   |

### 為什麼 HIGH 是亮、LOW 是滅?

板載 LED 的電路:

```
GPIO 2 ──→ 電阻 ──→ LED ──→ GND (0V)
```

- **寫 HIGH:** GPIO 2 = 3.3V，GND = 0V，兩端有 3.3V 落差 → 電流流過 LED → **亮**
- **寫 LOW:** GPIO 2 = 0V，GND = 0V，沒有落差 → 沒電流 → **滅**

就像水管兩端等高，水就不流。

> 冷知識:「HIGH = 亮」不是必然。有些板子(如許多 ESP8266)LED 反著接，寫 LOW 才亮，稱為低電位觸發(active-low)。哪天發現燈的行為跟程式相反，是硬體接法不同，不是程式壞了。

### 3.3V 是什麼概念?

| 東西             | 電壓                          |
| ---------------- | ----------------------------- |
| 一顆 AA 電池     | 1.5V                          |
| 兩顆 AA 電池串聯 | 3V ≈ **ESP32 的 HIGH (3.3V)** |
| 手機鋰電池       | 3.7V                          |
| USB / 行動電源   | 5V(板子的供電來源)            |
| 機車電瓶         | 12V                           |
| 台灣家用插座     | 110V(危險!)                   |

**3.3V ≈ 兩顆乾電池的力氣**，徒手摸完全沒感覺，玩 ESP32 很安全。

要記住的兩個數字:**ESP32 的世界是 3.3V，USB 供電是 5V。**
接任何模組前先確認電壓合不合 — 電壓不合，輕則不動作，重則燒晶片。

### 穩壓器 vs 電阻

USB 給的是 5V，但晶片只能吃 3.3V，中間靠板上的**穩壓器**降壓。它跟電阻的差別:

|          | 穩壓器                      | 電阻                 |
| -------- | --------------------------- | -------------------- |
| 水的比喻 | 智慧減壓閥                  | 捏細的水管           |
| 控制什麼 | **電壓**(水壓)              | **電流**(水流)       |
| 特性     | 主動調節，輸出永遠穩定 3.3V | 被動、固定，不會調整 |
| 在板子上 | 把 USB 5V 降成 3.3V 餵晶片  | 串在 LED 前限流保護  |

一句話:**穩壓器顧電壓，電阻顧電流。** 晶片要穩定的壓，LED 怕過大的流，兩者不能互相取代。
